## 二、核心能力

### 1. Tracing（追踪）
- 自动追踪 LLM 调用链
- 支持 framework：LangChain / LlamaIndex / OpenAI / Anthropic / Bedrock / Vertex / Cohere
- 支持装饰器：`@opik.track` 一行代码追踪
- 支持 OpenTelemetry 标准

### 2. Evaluation（评估）
- 自定义评估指标
- LLM-as-judge（用 LLM 评 LLM）
- 人类反馈收集
- A/B test / 实验对比
- 多种指标：relevance, hallucination, toxicity, etc.

### 3. Monitoring（监控）
- 实时 dashboard
- production traffic 监控
- alert / threshold 配置
- cost tracking

### 4. Datasets（数据集管理）
- 测试集版本控制
- 与 evaluation pipeline 联动
- golden dataset 自动构建

### 5. Playground
- 在线 LLM 实验场
- prompt 版本对比
- 多模型对比

## 三、5 分钟自部署（Docker）

```bash
# 1. 克隆仓库
git clone https://github.com/comet-ml/opik.git
cd opik/deployment

# 2. 启动
docker compose up -d

# 3. 访问 UI
open http://localhost:5173

# 4. Python SDK
pip install opik
opik configure
```

**资源需求**：
- 最小：4 GB RAM + 2 vCPU + 50 GB disk
- 推荐：8 GB RAM + 4 vCPU + 100 GB SSD
- 数据存储：PostgreSQL + ClickHouse + Redis

## 五、典型使用场景

### 场景 1：RAG 系统调试
```python
from opik import track, evaluate
from opik.evaluation.metrics import Hallucination, AnswerRelevance

@track
def my_rag_pipeline(query: str) -> str:
    context = retrieve(query)
    answer = generate(query, context)
    return answer

# 评估
experiment = evaluate(
    dataset=golden_dataset,
    task=my_rag_pipeline,
    metrics=[Hallucination(), AnswerRelevance()]
)
```

### 场景 2：Multi-Agent 监控
```python
from opik import track

@track(name="supervisor_agent")
def supervisor(query):
    plan = planner(query)
    return plan

@track(name="worker_agent")
def worker(task):
    result = executor(task)
    return result
```

### 场景 3：Production 监控
```python
from opik import opik_context

@track
def llm_call(prompt):
    response = client.messages.create(...)
    opik_context.update_current_span(
        metadata={"user_id": user_id, "cost": calculate_cost(response)}
    )
    return response
```
