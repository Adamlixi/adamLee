> 7 个 Comet 全调研完后的反直觉洞察 + 可执行建议。

## 7 个关键反直觉洞察

### 1. "Comet" 在 2026 已经是高频命名冲突
- 7 个互不相关的项目都叫 Comet
- 4 个独立公司选了 Comet 这个名字
- **搜索 "comet" 一定要带上下文**（Perplexity / ML / Chat / C# 等）
- **10 年后科技圈命名冲突只会更严重**

### 2. Perplexity Comet 不开源 ≠ 没有开源替代品
- 闭源 ≠ 护城河
- BrowserOS (12k★) + nanobrowser (13k★) + browser-use (105k★) 已经能复刻 90% Comet 能力
- **用户最终选 Perplexity Comet 是因为"品牌 + Perplexity 搜索 backend"**，不是技术
- 开源 AI 浏览器生态远比想象中成熟

### 3. Comet ML 公司是"开源 + SaaS"双引擎典范
- Opik (20k★, Apache 2.0) + litellm (53k★, MIT) 是开源层
- comet.com 实验管理 / 模型注册是 SaaS 层
- 2023 年战略转向 LLM observability 后，公司估值大概率翻倍
- **类比 GitLab / Elastic / MongoDB 路径**

### 4. litellm 是 Comet 公司"最被低估的资产"
- 53,972 stars (BerriAI/litellm)
- 比 Opik 还多 33k stars
- 2024 年被 Comet 收购时是整个 AI gateway 赛道的 leader
- 现在 Comet 公司估值大部分可能来自 litellm 生态
- **冷知识**：Comet 公司收购 BerriAI 后，原 BerriAI 仓库保留，comet-ml/litellm 只是个品牌镜像（2 stars）

### 5. dotnet/Comet 已经 archived，但被搜索时高频出现
- 1,673 stars + MIT license + dotnet org 名头
- 让人误以为活跃
- 实际 2022 年后无 commits
- **这是 OSS 生态的"僵尸项目"问题** — 看起来重要但已死

### 6. 开源 AI 浏览器的 license 策略 = 商业护城河
- **AGPL-3.0**：BrowserOS / Skyvern（防止 SaaS 化）
- **Apache 2.0**：nanobrowser / Steel.dev（鼓励企业采用）
- **MIT**：browser-use / Stagehand（鼓励一切）
- 选择 license 体现了对商业化的态度

### 7. 系统提示词泄露 ≠ 完整开源
- Perplexity Comet 提示词 2026-02 完整泄露
- 但泄露**只覆盖 AI 助手部分**
- 缺失：浏览器 shell / tab manager / Perplexity 搜索 backend / sync / 扩展系统
- **泄露 ≠ 复刻**，但让独立开发者可以"局部对标"

## 6 个可执行行动建议

### 如果你是用户（终端用户）
| 场景 | 建议 |
|---|---|
| 想要 Perplexity Comet 体验 + 免费 | 用 **BrowserOS**（AGPL, 12k★, macOS/Windows） |
| 想要 Chrome 扩展形式 + 多 agent | 用 **nanobrowser**（Apache 2.0, 13k★） |
| 想要隐私优先 + 完全本地 | 用 **Comet-AI (Electron)** 配 Ollama |

### 如果你是开发者
| 场景 | 建议 |
|---|---|
| 做 coding agent 要浏览器能力 | 用 **browser-use**（MIT, 105k★） |
| TypeScript first + 稳定 | 用 **Stagehand**（MIT, 23k★, Browserbase） |
| 企业级 + 高可靠性 | 用 **Skyvern**（AGPL, 22k★） |
| Serverless 部署 web agent | 用 **Notte**（2k★, serverless） |
| 给 AI agent 接 Chrome 能力 | 用 **chrome-devtools-mcp**（47k★, Google 官方） |

### 如果你是创业者 / 产品经理
| 场景 | 建议 |
|---|---|
| 想做"AI 浏览器"创业 | **别做浏览器**，做 AI agent SDK（赛道已饱和） |
| 想做"LLM observability" | 直接 fork **Opik**（Apache 2.0）二次开发 |
| 想做"LLM gateway" | 直接用 **litellm**（已有 53k stars 护城河） |
| 想做"AI 浏览器扩展" | 看 **nanobrowser** 仓库结构，差异化定位 |

### 如果你是数据科学家
| 场景 | 建议 |
|---|---|
| 传统 ML 实验追踪 | 用 **comet_ml SDK**（pip install）+ comet.com SaaS |
| LLM 应用 eval + obs | 用 **Opik**（完全开源） |
| RAG 系统 debug | 用 **Opik + LangChain integration** |
| 团队协作 + dashboard | 订阅 comet.com（$$$） |

### 如果你是企业架构师
| 场景 | 建议 |
|---|---|
| 内部 AI 应用 obs 平台 | **自部署 Opik**（Apache 2.0） |
| 内部 LLM API 网关 | **自部署 litellm**（Apache 2.0） |
| AI agent 浏览器自动化 | **评估 Skyvern 企业版**（商业 license） |
| LLM 成本控制 | **用 litellm 的 cost tracking** + OpenTelemetry 导出 |

## 5 个值得长期跟踪的信号

1. **Perplexity Comet 是否开源**：未来 12 个月内极小概率（< 5%），但 Perplexity 上市压力会增加可能性
2. **Opik 是否变更 license**：参考 HashiCorp Terraform 事件（2024-08），关注 2026-2027
3. **Comet ML 是否 IPO / 被收购**：SaaS + 开源模式跑通后，IPO 概率提升
4. **litellm 是否独立拆出来**：53k stars 太大了，Comet 母公司可能稀释
5. **getcomet.net 是否复活**：Reddit 替代赛道有反复热潮可能

---

## 6 个关键风险

1. **AGPL-3.0 法律风险**：BrowserOS / Skyvern 不能直接 SaaS 化（除非开源你的代码）
2. **公司策略变化风险**：Comet ML 公司可能改 license（参考 HashiCorp）
3. **维护风险**：dotnet/Comet 已死，getcomet.net 已下线
4. **Stars 虚高风险**：很多项目 "Used by 21k" 不是真实 forks
5. **Perplexity 战略风险**：Comet 团队被重组 / 砍掉可能
6. **AI Agent 赛道剧烈变动**：6-12 个月可能出来新范式（MCP 已是新爆款）
